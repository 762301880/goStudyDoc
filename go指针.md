## 🌟 一句话先理解指针

👉 指针 = “某个东西的地址”

而不是东西本身。

## 先看 Go 里的普通变量

```go
a := 10
```

此时：

- `a` = 10（真实的值）

## 指针版本

```go
a := 10
p := &a
```

这里：

- `&a` = a 的地址
- `p` = 指向 a 的“指针”

### 🔍 你可以这样理解：

| 写法 | 含义           |
| ---- | -------------- |
| a    | 真实值 10      |
| &a   | a 的地址       |
| *p   | 通过地址找到值 |

🔧 举个最关键例子

```
a := 10
p := &a

fmt.Println(*p) // 10
```

👉 `*p` 就是“顺着地址找到值”

# 🚀 那 Gin 里的 `*gin.Context` 是什么？

```
func Login(c *gin.Context)
```

这里：

👉 `c` 是 **Context 的指针**

# 💡 为什么 Gin 一定要用指针？

因为 Context 很大：

它里面有：

- 请求参数
- Header
- JSON body
- 响应方法
- 中间件数据
- 连接状态

👉 如果不用指针，每次调用函数都会**复制一份 Context（很慢 + 很浪费内存）**

##  如果不用指针会怎样？

```
func Login(c gin.Context) // ❌ 假设这样
```

问题：

- 每次请求复制一整份 Context
- 性能差
- 修改不会影响外部

## 用指针的好处

```
func Login(c *gin.Context)
```

✔ 不复制（只传地址）
 ✔ 可以直接修改同一个 Context
 ✔ Gin 官方设计就是这样

# 用一句超直觉比喻

👉 `gin.Context` 就像一份“订单”

### 不用指针：

你每次都复印一份订单（浪费纸 + 不同步）

### 用指针：

你拿的是同一份订单的“编号”（地址）
